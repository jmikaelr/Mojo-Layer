#+TITLE: Mojo Layer
#+TAGS: layer|lang|mojo

* Description
This layer adds support for the [[https://www.modular.com/mojo][Mojo 🔥]] programming language.

** Features
- Syntax highlighting (keywords, types, decorators, numbers, triple-quoted strings)
- Indentation (block openers, dedenters, paren alignment, multiline signatures)
- LSP integration via lsp-mode's built-in Mojo client (=mojo-lsp-server=)
- Flycheck syntax checking via =mojo build=, pixi-aware
- Go-to-definition: LSP when available, ripgrep text-search fallback
- Find references (LSP / ripgrep)
- =mojo format= integration (buffer, region, file, project)
- REPL with send region/buffer/line/defun helpers
- Build, run, test, and clean commands — all pixi-aware
- Projectile project type registration
- YASnippet snippets
- Evil text objects for functions (=vif= / =vaf=)
- Imenu and which-func integration

* Install
** Layer
Add =mojo= to =dotspacemacs-configuration-layers=:

#+begin_src emacs-lisp
(setq-default dotspacemacs-configuration-layers '(mojo))
#+end_src

** Prerequisites
- [[https://www.modular.com/mojo][Mojo]] installed, available as =mojo= on =PATH= or via [[https://prefix.dev/][pixi]]
- =mojo-lsp-server= on =PATH= for LSP features (installed with Mojo via pixi)
- =rg= ([[https://github.com/BurntSushi/ripgrep][ripgrep]]) for fallback definition and reference search

* Configuration
** Variables

| Variable                        | Description                                     | Default                          |
|---------------------------------+-------------------------------------------------+----------------------------------|
| =mojo-format-on-save=           | Format buffer on save with =mojo format=        | =nil=                            |
| =mojo-indent-offset=            | Indentation step in spaces                      | =4=                              |
| =mojo-use-project-pixi=         | Run Mojo commands through project pixi          | =t=                              |
| =mojo-use-global-pixi=          | Fall back to =~/.pixi= before system PATH       | =t=                              |
| =mojo-global-pixi-env-priority= | Preferred env names under =~/.pixi/envs=        | =("mojo" "default")=            |
| =mojo-pixi-no-progress=         | Pass =--no-progress= to =pixi run=              | =nil=                            |
| =mojo-build-arguments=          | Extra args for =mojo build=                     | =()=                             |
| =mojo-run-arguments=            | Extra args for =mojo run=                       | =()=                             |
| =mojo-run-project-entrypoint=   | Explicit run entrypoint (relative to root)      | =nil= (auto-detect)             |
| =mojo-run-entrypoint-candidates=| Candidates for auto-detecting run entrypoint    | =("main.mojo" "src/main.mojo")= |
| =mojo-test-arguments=           | Extra args for test runs                        | =()=                             |
| =mojo-test-project-target=      | Explicit test target (relative to root)         | =nil= (auto-detect)             |
| =mojo-test-target-candidates=   | Candidates for auto-detecting test target       | =("tests" "test")=              |
| =mojo-test-add-project-include= | Pass =-I <project-root>= on test runs           | =t=                              |
| =mojo-clean-command=            | Custom clean command (=nil= uses =pixi clean=)  | =nil=                            |
| =mojo-stdlib-path=              | Stdlib source root for ripgrep fallback search  | =nil=                            |
| =mojo-extra-source-paths=       | Extra source roots for ripgrep fallback search  | =nil=                            |
| =mojo-repl-arguments=           | Args passed to =mojo= when starting the REPL    | =("repl")=                      |

** Pixi command resolution
When =mojo-use-project-pixi= is non-nil, Mojo CLI commands resolve as:
1. =pixi run [--no-progress] mojo= from project root (preferred — sets up environment)
2. project-local =.pixi/envs/*/bin/mojo=
3. global =~/.pixi= tool paths (when =mojo-use-global-pixi= is non-nil)
4. system =PATH=

** LSP
LSP uses lsp-mode's built-in Mojo client, which finds =mojo-lsp-server= via
=exec-path= (your shell =PATH=). No extra configuration is needed if
=mojo-lsp-server= is installed.

** Ripgrep fallback navigation
=gD= (force fallback) and =SPC m g G= search for definitions using ripgrep
across the project root and any configured source paths. To also search inside
the Mojo stdlib source tree, clone it and set:

#+begin_src emacs-lisp
(mojo :variables
      mojo-stdlib-path "/path/to/mojo/stdlib/src")
#+end_src

** Example configuration

#+begin_src emacs-lisp
(setq-default dotspacemacs-configuration-layers
  '((mojo :variables
          mojo-format-on-save t
          mojo-use-project-pixi t
          mojo-stdlib-path "~/src/mojo/stdlib/src"
          mojo-run-project-entrypoint "src/main.mojo"
          mojo-test-arguments '("--warn-on-unstable-apis"))))
#+end_src

** Notes
- =mojo test= was removed in late 2025. Tests run via =mojo run=.
- Mojo evolves rapidly; keyword/type highlighting tracks the current stdlib API.
